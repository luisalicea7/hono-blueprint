# chefsito-backend — Claude Guidelines

## Stack

- **Runtime:** Node.js + TypeScript (`NodeNext` module, `strict: true`, `verbatimModuleSyntax: true`)
- **Framework:** Hono (`app.ts` exports the app, `index.ts` starts the server)
- **Auth:** Better Auth — email/password, admin, username, openAPI, i18n (EN/ES) plugins
- **Database:** PostgreSQL via Drizzle ORM
- **Cache:** Redis via ioredis — used for Better Auth session storage AND query caching (flag-gated)
- **Validation:** Zod v4
- **Logging:** pino + pino-pretty
- **Tests:** Vitest — integration only, sequential (`fileParallelism: false`)

## Conventions

### Error Handling

- **Services:** wrap all DB calls in try/catch — catch, log with pino, re-throw as `AppError` subclass (`InternalError`) for unexpected errors, let `AppError`s pass through with `if (isAppError(err)) throw err`
- **Controllers:** single try/catch per method — check `isAppError(err)` for typed status, log unknown errors, return 500
- **`AppError` subclasses:** `UnauthorizedError` (401), `NotFoundError` (404), `ForbiddenError` (403), `BadRequestError` (400), `ConflictError` (409), `InternalError` (500), `ServiceUnavailableError` (503)

### Controllers

- Extract `user` from context with `c.get("user")!` — `requireAuth` guarantees it's non-null
- Parse request body with Zod schema before passing to service
- Parse query params with Zod schema for list endpoints

### Services

- Always receive `userId` and scope every DB query to it
- Verify ownership before any update/delete (findFirst → throw NotFoundError if missing)
- Validate foreign key ownership before insert
- Cache reads at the top of each read function; bust cache after every mutation

### Caching

- Controlled by `CACHE_ENABLED` env var (default `false`)
- Cache keys include env prefix: `cache:{NODE_ENV}:{userId}:{domain}:...`
- `getCached` / `setCached` fail silently (warn log) — never break a request due to cache errors

### Schemas

- Zod schemas live in `src/schemas/` — named `*Schemas.ts` (e.g. `generationSchemas.ts`)
- Types live in `src/types/` — never inline complex return types in service files

### Routes

- All app routes live under `/v1` prefix
- Auth routes remain at `/api/auth/*` (Better Auth convention)
- `sessionMiddleware` applied globally in `app.ts`
- `requireAuth` applied per-route or globally in route groups
- Health routes are public (`/v1/health/*`)

### Testing

- Integration tests use `app.request()` directly — no HTTP server needed
- Tests run sequentially (`fileParallelism: false`, `maxConcurrency: 1`)
- Test timeout: 15 seconds
- `afterEach` cleans up with `deleteUserByEmail` — cascade deletes all user data
- `signUpAndSignIn` helper creates a fresh user with random UUID email each test
- Tests assert 400 for Zod validation errors, 401 for unauthenticated, 404 for not found

### Database

- All tables use `uuid` PKs generated by `pg_catalog.gen_random_uuid()`
- Soft relationships use `onDelete: "set null"`
- Hard relationships use `onDelete: "cascade"`
- Updated-at fields use `$onUpdate(() => new Date())`
