# hono-blueprint

Production-ready Hono + TypeScript backend scaffold. Auth, database, caching, logging, and error handling wired up out of the box.

## Stack

| Layer            | Choice                                                                         |
| ---------------- | ------------------------------------------------------------------------------ |
| Runtime          | Node.js + TypeScript (`NodeNext`, `strict`)                                    |
| Framework        | [Hono](https://hono.dev)                                                       |
| Auth             | [Better Auth](https://better-auth.com) — email/password, admin, username, i18n |
| Database         | PostgreSQL via [Drizzle ORM](https://orm.drizzle.team)                         |
| Cache / Sessions | Redis via [ioredis](https://github.com/redis/ioredis)                          |
| Validation       | [Zod v4](https://zod.dev)                                                      |
| Logging          | [pino](https://getpino.io) + pino-pretty                                       |
| Tests            | [Vitest](https://vitest.dev) — integration only                                |

## Getting Started

### Prerequisites

- Node.js 20+
- PostgreSQL
- Redis
- [Bun](https://bun.sh) (or npm / pnpm)

### Setup

```bash
# Install dependencies
bun install

# Copy environment variables
cp .env.example .env
# Fill in .env

# Run migrations
bun run db:migrate

# Start dev server
bun run dev
```

Server starts at `http://localhost:3000`.

## Environment Variables

```text
PORT                  # default 3000
NODE_ENV              # development | test | production
APP_NAME              # default chefsito-backend
DATABASE_URL          # PostgreSQL connection string
REDIS_URL             # Redis connection string
BETTER_AUTH_SECRET    # Auth signing secret
BETTER_AUTH_URL       # Base URL for auth (used for cookies)
CORS_ORIGINS          # Comma-separated list (default http://localhost:5173 in dev)
CACHE_ENABLED         # true | false (default false)
```

## Project Structure

```text
src/
  app.ts              # Hono app
  index.ts            # Server startup + graceful shutdown
  config/             # Zod-validated env config
  lib/
    auth.ts           # Better Auth setup
  database/
    index.ts          # Drizzle client
    schema.ts         # Tables, enums, relations
    migrations/       # Generated by drizzle-kit
  cache/
    redis.ts          # ioredis client
    cacheService.ts   # getCached, setCached, invalidatePrefix
  schemas/            # Zod schemas — one file per domain
  types/              # Shared TypeScript types
  middleware/
    validateAuth.ts   # sessionMiddleware, requireAuth, requireAdmin
  utils/
    logger.ts         # pino instance
  routes/
    index.ts          # Mounts all routers
    v1/
      health.ts       # /v1/health
  controllers/v1/
  services/
test/
  integration/
```

## Scripts

```bash
bun run dev           # Dev server with watch
bun run build         # Compile TypeScript
bun run start         # Run compiled output
bun run typecheck     # Type check without emit
bun run test          # Run integration tests
bun run db:generate   # Generate Drizzle migrations
bun run db:migrate    # Apply migrations
bun run db:studio     # Open Drizzle Studio
bun run format        # Format with Prettier
```
